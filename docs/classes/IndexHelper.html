ensureIndex

  • ensureIndex(options): void

  • Ensures that future documents inserted or updated using this client instance will be indexed according to the given attribute, provided that they contain that attribute. Compound indexes can be specified by providing an array for attribute.

    Queries may be performed using compound indexes without specifying all attributes in the compound index so long as there is at least one value (or the attribute name for "has" queries) specified for consecutive attributes starting with the first. This allows for querying using only a prefix of a compound index. However, uniqueness will not be enforced unless all attributes in the compound index are present in a document.

      The options to use.
